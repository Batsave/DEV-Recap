## Qu'est-ce que Git ?

**Git** est un système de contrôle de version distribué.  
Il permet de suivre l’historique des fichiers, collaborer à plusieurs et revenir à un état précédent en cas d’erreur.  

➡ Contrairement à un simple backup, Git enregistre chaque modification avec un **commit**.

---

## Git vs GitHub

- **Git** = l’outil installé en local, pour gérer tes dépôts.  
- **GitHub/GitLab/Bitbucket** = plateformes en ligne pour héberger et collaborer sur les dépôts.  

---

## Exemple rapide

```bash
git init             # créer un dépôt Git
git add .            # ajouter tous les fichiers
git commit -m "Init" # valider les fichiers
```

---

## Bonnes pratiques

- Toujours écrire des **messages de commit clairs**.  
- Ne pas confondre **Git (outil)** et **GitHub (plateforme)**.  
- Penser à `.gitignore` pour exclure les fichiers inutiles.
