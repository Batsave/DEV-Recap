## Bonnes pratiques générales

- Commiter souvent mais avec des changements cohérents.  
- Utiliser des messages de commit clairs :  
  - Mauvais : "fix"  
  - Bon : "Corrige l'affichage du header sur mobile"  
- Toujours créer une branche pour une nouvelle fonctionnalité.  
- Vérifier `git status` et `git diff` avant de commiter.  
- Utiliser `.gitignore` pour éviter de versionner des fichiers inutiles.
